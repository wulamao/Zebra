设计初衷
    事件信息可以触发注册在该事件下的任务
    支持不同种类的任务（不限定回调函数的类型）

    事件应该能够传递参数

    存储事件的结构应该抽出来，方便更换数据结构
        考虑任务的添加、删除、查询、排序

    事件类型的存储结构使用队列

事件结构体类型定义
    事件类型 id unsigned int  
    事件优先序列 priority unsigned int
    事件回调函数缓存地址 cache_addr void **
    事件传递参数 arg void *

具体的事件类型应该有单独的头文件管理

事件结构与事件管理结构

事件的行为
    事件队列初始化 init
    事件添加 register
    事件删除 remove
    事件查找 lookup
    事件排序 sort
